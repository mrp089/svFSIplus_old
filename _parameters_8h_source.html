22 /// @brief The Parameter class template is used to store a named
23 /// paramater and its scalar value as a basic type: bool, double,
24 /// int and string.
25 ///
26 /// The classes defined here are used to process svFSIplus simulation parameters read in
27 /// from an Extensible Markup Language (XML) format file. XML is a simple text-based format
28 /// for representing structured information.
29 ///
30 /// An XML document is formed as an element tree. The XML tree starts at a root element and
31 /// branches from the root to sub-elements. All elements can have sub-elements:
32 ///
33 /// \code{.cpp}
34 /// <svFSIFile>
35 /// <element>
36 /// <subelement>.....</subelement>
37 /// </element>
38 /// </svFSIFile>
39 /// \endcode
40 ///
41 /// The elements in the svFSIplus simulation file are represented by sections of
42 /// related parameters. Sub-elements are refered to as sub-sections.
43 ///
44 ///-----------------
45 /// Parameters class
46 ///-----------------
47 /// The Parameters class is the top level class. It contains objects used to store
48 /// parameters for the sections making up an XML simulation parameters file
49 ///
50 /// 1) General (GeneralSimulationParameters)
51 /// 2) Mesh (MeshParameters)
52 /// 3) Equation (EquationParameters)
53 /// 4) Projection (ProjectionParameters)
54 ///
55 /// Each object contains methods to parse the XML file for the parameters defined for it.
56 /// These section objects may also contain objects representing the sub-sections defined
57 /// for each section.
58 ///
59 ///-----------------
60 /// Section objects
61 ///-----------------
62 /// Each section object contains objects representing parameters. A parameter's name and value
63 /// is stored using either a Parameter and VectorParamater template objects. A parameter
64 /// value is stored as a basic type: bool, double, int and string.
65 ///
66 /// Parameter objects in a section class are named using the same XML element name with the 1st
67 /// character lower case.
68 ///
69 /// Example: GeneralSimulationParameters class
70 ///
71 /// \code{.cpp}
72 /// Parameter<bool> verbose; // <Verbose>
73 /// Parameter<double> spectral_radius_of_infinite_time_step; // <Spectral_radius_of_infinite_time_step>
74 /// Parameter<double> time_step_size; // <Time_step_size>
75 /// \endcode
76 ///
77 /// The name and default value for each parameter is defined in a section object's constructor.
78 ///
79 /// Parameter values are set using the set_values() method which contains calls to tinyxml2
80 /// to parse parameter values from an XML file.
81 ///
82 /// Each section object inherits from the ParameterLists class. This class provides methods to
83 /// store parameters in a map and process iterated over them for setting values and other operations.

153  std::string name_ = "";
154  bool required_ = false;

325  /// @brief Check if any required parameters have not been set.
326  void check_required()
327  {
328  bool unset_found = false;
329 
330  for (auto& [ key, param ] : params_map) {
331  if (std::visit([](auto&& p) {
332  return !p->check_required_set();
333  }, param)) {
334  throw std::runtime_error(xml_element_name + " XML element '" + key + "' has not been set.");
335  }

339  /// @brief Get the defined parameters as a map of strings.
340  std::map<std::string,std::string> get_parameter_list()
341  {
342  std::map<std::string,std::string> params;
343 
344  for (auto& [ key, param ] : params_map) {
345  std::visit([&params](auto&& p) {
346  params[p->name()] = p->svalue();
347  }, param);
348  }
349 
350  return params;
351  }

363  }
